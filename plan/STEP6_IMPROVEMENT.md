## 提案1：週足トレンドフィルタの追加（即時対応・最優先）

### 根拠となるデータ

強気シグナルを「週足トレンドの方向」と「RSI水準」で分解した結果：

| 週足トレンド | RSI < 45 | 件数 | 勝率 |
|---|---|---|---|
| 上昇 (+1) | No（45以上） | 25件 | 64.0% |
| 上昇 (+1) | **Yes（45未満）** | **3件** | **100.0%** |
| 下降 (-1) | No（45以上） | 15件 | 33.3% |
| 下降 (-1) | Yes（45未満） | 21件 | 61.9% |

**週足が下降トレンド × RSI45以上の強気シグナルの勝率が33.3%**と最も低く、
これが2026年の崩壊の実体です。逆に「週足上昇 × RSI45未満」は100%（サンプル3件）。

### 候補フィルタ条件の単独効果

| 条件 | 件数 | 勝率 | 平均リターン |
|---|---|---|---|
| 条件なし（現状） | 64件 | 57.8% | 0.44% |
| 週足上昇のみ | 28件 | 67.9% | 0.97% |
| RSI < 45のみ | 24件 | 66.7% | 1.03% |
| ATRパーセンタイル < 0.75 | 36件 | 66.7% | 1.05% |
| **週足上昇 × RSI<50 × ATR<0.75** | **5件** | **80.0%** | **1.80%** |

件数は絞られるが、組み合わせることで勝率・リターンが大幅に向上する。

### 実装方法

`src/step5_assist_signal.py` の `generate_assist_signal()` に条件を追加します。

## 提案2：ドローダウン予測モデルのターゲット変更（中期対応）

### 現在のターゲットが難しい理由

`avoid_entry_flag`（10日以内に3%のDD）の発生率は**55%**と過半数であり、
クラスが均等すぎて「何が危険か」の識別が困難です。
また前回のレポートが示すようにMLモデルが逆学習しやすい状態になっています。

### 代替ターゲットの候補

実データのATRバンド別勝率から、より予測しやすい目標を定義できます：

```
ATRパーセンタイル帯別 勝率（全データ）:
  低(〜25%):    53.6%  ← 安定
  中低(25-50%): 61.3%  ← 最も良好
  中高(50-75%): 41.5%  ← 苦手
  高(75-90%):   40.4%  ← 苦手
  過熱(90%+):   46.2%  ← ATRフィルタが既に対応
```

「中高〜高ボラティリティ帯（ATRパーセンタイル 0.50〜0.90）での成績が悪い」という
明確なパターンがあるため、ターゲットを以下に変更することで識別しやすくなります：

```python
# 変更案: 5日後の最大下落幅が一定以上かどうかをターゲットにする
# avoid_entry_flag（10日/3%）→ high_risk_flag（5日/2%）

df["high_risk_flag"] = (df["forward_max_drawdown_5d"] <= -0.02).astype(int)
# 発生率が下がる分、クラスのアンバランスが生まれ識別しやすくなる
```

また `antigravity_2606240844.md` では「Step4 ROC-AUC < 0.55 の銘柄はMLフィルタを
自動無効化する」という提案がありましたが、これを `src/step6_filter.py` に自動判定として
組み込む方法が現実的です：

```python

        results = {"oos_predictions": pd.Series(-1.0, index=step3_df.index)}
```

## 提案3：フィルタ逆相関の自動検知（中期対応）

前回のレビューで最も重大だった「フィルタが逆方向に機能している」問題を
**実行時に自動検知して警告を出す**仕組みを追加します。

### 実装方法

```python
# src/step6_filter.py に追加する検証関数
def validate_filter_direction(df: pd.DataFrame) -> None:
    """
    dd_prob と tb_label の相関を確認し、
    逆相関（フィルタが逆機能）の場合に警告を出す。
    """
    from scipy.stats import spearmanr
    
    # dd_probが有効（0でも-1でもない）な行のみ対象
    valid = df[df["drawdown_prob"].between(0.01, 0.99)]
    if len(valid) < 20:
        print("[Step6 警告] dd_prob の有効サンプルが少なすぎて検証できません。")
        return

    corr, pval = spearmanr(valid["drawdown_prob"], valid["tb_label"])
    
    print(f"\n[Step6 フィルタ方向検証]")
    print(f"  dd_prob と tb_label のスピアマン相関: {corr:.3f}  p値: {pval:.3f}")
    
    if corr > 0.05:
        print("  ✅ 正常: dd_probが高いほど勝率が低い（フィルタが正方向に機能）")
    elif corr < -0.05 and pval < 0.1:
        print("  ⚠️  警告: dd_probが高いほど勝率が高い（フィルタが逆機能しています）")
        print("     → --drawdown-prob-limit 1.0 での実行を推奨します。")
    else:
        print("  ℹ️  中立: dd_probと勝率の相関が弱く、フィルタの効果が不明確です。")
```

## 提案4：相場フェーズ別の切り替えロジック（中期対応）

2026年の崩壊は週足下降トレンドへの移行が引き金でした。
相場フェーズ（上昇/下降）に応じてロジックを切り替えることで対応できます。

### フェーズ判定と推奨戦略

| 週足トレンド | RSI水準 | 推奨戦略 | シグナル例 |
|---|---|---|---|
| 上昇 (+1) | < 50（調整中） | 順張り押し目買い | **強気を積極採用** |
| 上昇 (+1) | ≥ 70（過熱） | 様子見 | 警戒を優先 |
| 下降 (-1) | < 35（売られすぎ） | 逆張り慎重エントリー | RSI < 35のみ強気 |
| 下降 (-1) | ≥ 35 | **エントリー禁止** | 強気シグナルでも見送り |

```python
# src/step5_assist_signal.py への実装案
def _apply_phase_filter(d: pd.DataFrame) -> pd.Series:
    """
    相場フェーズに応じて強気シグナルをフィルタする。
    週足下降 × RSI35以上 の強気は中立に格下げする。
    """
    is_bullish = d["assist_signal"] == "強気"
    downtrend = d["weekly_trend"] == -1
    rsi_not_oversold = d["rsi14"] >= 35

    # 下降トレンド × RSI売られすぎ圏でない = 強気を中立に格下げ
    downgrade = is_bullish & downtrend & rsi_not_oversold
    
    signal = d["assist_signal"].copy()
    signal[downgrade] = "中立"
    return signal
```
