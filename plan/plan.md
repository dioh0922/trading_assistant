# AIトレードシステム構築：実装ステップガイド

[cite_start]本ガイドは、日足データに環境認識（マルチタイムフレーム）とドメイン知識を加えることで、予測精度と運用の実用性を高めるためのロードマップです [cite: 16]。

---

## ステップ1：データ収集とマルチタイムフレームの統合
[cite_start]日足のみの学習から脱却し、環境認識のための情報を追加します [cite: 1, 9, 16]。

* [cite_start]**データセットの作成**: 日足データに加え、週足データを同一期間で揃えて統合します [cite: 9, 16]。
* **マルチタイムフレーム指標の作成**:
    * [cite_start]週足の移動平均線の傾き（トレンド方向の判定）を追加します [cite: 1, 8, 9, 16]。
    * [cite_start]日足のボラティリティ（ATR）やボリンジャーバンドの％Bを計算します [cite: 2, 4, 9, 16]。
* [cite_start]**intraday情報の集約**: 日中の高値・安値の比率（ヒゲの長さ）など、日足の枠を超えた情報を特徴量に加えます [cite: 1, 9, 16]。

---

## ステップ2：特徴量エンジニアリング（ドメイン知識の注入）
[cite_start]生の価格データではなく、市場の「状態」を表す数値に変換します [cite: 2, 4, 10, 17]。

* **相対指標の生成**:
    * [cite_start]移動平均線からの乖離率で「下げすぎ・上げすぎ」を定量化します [cite: 4, 10, 17]。
    * [cite_start]RSIで「強気のダイバージェンス」を検出するロジックを実装します [cite: 2, 4, 10, 17]。
* **過熱感の数値化**:
    * [cite_start]ATRに対する乖離幅を算出し、直近の急騰が異常かどうかを判定します [cite: 2, 4, 10, 17]。
    * [cite_start]価格変化の加速度（モメンタム）を計算し、急騰の一時性を予測対象に含めます [cite: 4, 10, 17]。

---

## ステップ3：ターゲット（ラベル）の設計
[cite_start]「上がったか下がったか」ではなく「特定の期間における期待値」を定義します [cite: 2, 5, 10, 18]。

* [cite_start]**トリプルバリア法の適用**: 「利確目標」「損切りライン」「期間」を設定し、どの境界に先に触れるかを予測させます [cite: 2, 10, 18]。
* **ラベルの分類**:
    * [cite_start]**買いシグナル用**: 損切りにならず、かつ利確目標に到達した場合を「1」とします [cite: 5, 10, 18]。
    * [cite_start]**見送りシグナル用**: エントリー後に一定割合（例：3%）のドローダウンが発生するかどうかを予測させます [cite: 5, 10, 18]。

---

## ステップ4：モデル構築とバリデーション
[cite_start]時系列データの特性を考慮した学習を行います [cite: 3, 11, 18]。

* [cite_start]**アルゴリズム選定**: 構造化データに強い LightGBM または XGBoost の採用を推奨します [cite: 3, 11, 18]。
* [cite_start]**Walk-Forward Validation**: データのシャッフルは行わず、過去のデータで学習し、常に未来のデータで検証を行う時系列バリデーションを徹底します [cite: 3, 11, 18]。

## ステップ5：アシストロジックの実装（3段階シグナル）
[cite_start]最終的にAIの出力を解釈可能な形式に落とし込みます [cite: 5, 11, 19]。

| シグナル | 状態 | 判断の考え方 |
| :--- | :--- | :--- |
| **強気** | 保持・新規買い | [cite_start]RSI反転 ＋ 移動平均線乖離の縮小 [cite: 6, 13, 20] |
| **警戒** | エントリー見送り | [cite_start]ATRに対する価格乖離が異常値（急騰） [cite: 7, 13, 20] |
| **中立** | 様子見 | [cite_start]上記条件を満たさない局面 [cite: 7, 14, 21] |
