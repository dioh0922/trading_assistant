# AIトレードシステム 成果物の読み方ガイド

このレポートは、株式投資にあまり詳しくない方向けに、各ステップで出力される
グラフやデータ（CSV）を**どう読めばいいか**をやさしく解説するものです。

実行する銘柄やデータの期間が変わっても、この読み方の基本は変わりません。
「今回はこうだった」という個別の結果ではなく、「グラフのここを見れば何がわかるか」
という汎用的な視点でまとめています。

---

## そもそも何をやっているプロジェクトなのか

「この株は今、買い時なのか・様子見すべきなのか」をAIに判断させようとしています。
ただし、いきなりAIに「上がるか下がるか当ててみて」と頼んでもうまくいきません。
そこで、

1. 株価の動きを「人間にもわかる指標」に変換し（ステップ1〜2）
2. 「過去に同じような状況の時、実際にどうなったか」を答え合わせデータとして用意し（ステップ3）
3. AIにその答え合わせデータを使って学習させ（ステップ4）
4. 最後にAIの判断を「強気・警戒・中立」という人間にわかりやすい3段階の信号に変換する（ステップ5）

という流れで進めています。

---

## ステップ1：株価データの土台づくり（`step1_chart.png`）

### 何をしているか
日々の値動き（日足）だけでなく、1週間単位の値動き（週足）も合わせて見ることで、
「短期的にはバタついていても、大きな流れでは上昇中」のような**大きな波**を把握できるようにしています。

### グラフの読み方

| パネル | 見方 |
|---|---|
| ① 価格＋移動平均線＋背景色 | オレンジ線（25日平均）・緑線（75日平均）より価格が上にあれば短期的に強い状態。背景が薄い赤＝週足ベースで上昇トレンド、薄い青＝下降トレンド |
| ② 出来高 | 棒グラフが急に伸びている日＝その日に多くの売買があった（注目された日） |
| ③ RSI | 0〜100の数値。70を超えると「買われすぎ」、30を下回ると「売られすぎ」の目安 |
| ④ ボリンジャー%B | 1を超えると価格が普段の値動きの範囲を超えて上に飛び出している（過熱気味）、0を下回ると下に飛び出している |
| ⑤ ATR | その時々の値動きの荒さ（ボラティリティ）。数値が大きいほど値動きが荒い相場 |
| ⑥ モメンタム加速度 | プラスなら値上がりの勢いが増している、マイナスなら勢いが衰えている |

### チェックポイント
- 価格が急騰・急落している局面では、ATR（⑤）も一緒に跳ね上がっていないか確認しましょう。**値動きが大きい時ほど、値動き自体も荒くなりやすい**という関係がよく見られます。
- 背景色（赤/青）と実際の価格の動きが一致しているかを見ると、「週足の大きな流れ」と「日々の値動き」がズレている局面（=変化の兆し）を見つけやすくなります。

## ステップ2：相場の「過熱具合」を数値化（`step2_chart.png`）

### 何をしているか
ステップ1の指標だけだと「なんとなく買われすぎ」程度しかわかりません。
ステップ2では、それを**統計的にどれくらい異常な状態か**まで数値化します。

### グラフの読み方

| マーカー・パネル | 意味 |
|---|---|
| 緑の▲（強気ダイバージェンス） | 株価は下がっているのに、RSI（勢いの指標）は上がり始めている状態。**「下落の勢いが弱まってきたサイン」**としてよく使われる |
| 赤の▼（一時的急騰の疑い） | 短期間で急激に値上がりしていて、それが長続きしない可能性が高い状態 |
| オレンジ・青の点（過熱マーカー） | 通常の値動きの範囲を大きく超えている日 |
| MA乖離Zスコア | 移動平均線からどれくらい離れているかを、過去60日の自分自身の値動きと比べて「±2を超えたら異常値」という基準で表示 |
| ATR正規化乖離 | 値動きの荒さ（ATR）を基準にして、今の価格がどれくらい飛び出しているかを表す |

### チェックポイント
- 緑の▲（強気ダイバージェンス）が**価格の下落局面の終盤付近に集中しているか**を見ると、このサインが実際に反転のタイミングを捉えられていたかを目視で確認できます。
- 過熱マーカー（オレンジ・青の点）が全体の何割くらいの日に出ているかをざっくり数えると、その銘柄・期間が「普段から値動きが荒いタイプ」かどうかの目安になります。割合が多いほどハイボラティリティな相場だったと言えます。

## ステップ3：「もし買っていたら結果はどうだったか」を答え合わせ（`step3_chart.png`）

### 何をしているか
これが一番重要なステップです。**「もしこの日に買っていたら、その後どうなったか」**を
過去のデータを使って機械的にシミュレーションしています。

具体的には、それぞれの日について

- 「これくらい上がったら利益確定」というライン（利確ライン）
- 「これくらい下がったら損切り」というライン（損切りライン）
- 「最大でも10営業日（約2週間）以内に決着をつける」というルール

を設定し、**利確ラインと損切りライン、どちらに先に触れたか**を記録しています。これを
「トリプルバリア法」と呼びます。

### グラフの読み方

| パネル | 見方 |
|---|---|
| ① 価格＋ラベルマーカー | 緑▲＝利確ラインに到達した日（「買って成功」だった日）、赤▼＝利確に届かなかった日 |
| ② リターン分布 | 実際に利確/損切りした時の損益（%）の分布。山が2つに割れていれば「利確で終わった集団」と「損切りで終わった集団」がはっきり分かれていることを意味する |
| ③ バリア到達内訳（円グラフ） | 全体のうちどれくらいが利確・損切り・期間切れで終わったか |
| ④ フォワード最大ドローダウン | エントリー後、最大でどれくらい含み損を抱える局面があったか |
| ⑤ 見送りフラグ | 黒い帯＝その日買うと、保有中に一定割合以上の含み損を抱える可能性が高かった期間 |

### チェックポイント
- 円グラフ（③）の「利確到達」割合が著しく低ければ、利確ライン・損切りラインの設定（広さ）が
  その銘柄の値動きに合っていない可能性があります。極端に偏っている場合はパラメータの見直しを検討しましょう。
- 見送りフラグ（⑤）が立っている期間の割合が高いほど、その銘柄・期間は「保有中に含み損を抱えやすい」
  リスクの高い局面が多かったことを意味します。

## ステップ4：AIにパターンを学習させる（`step4_chart.png`）

### 何をしているか
ステップ3で作った「答え合わせデータ」を使って、AI（機械学習モデル）に
「こういう状態の時は利確に到達しやすい／しにくい」というパターンを学習させます。

学習には**LightGBM**（または代替としてscikit-learnの勾配ブースティング）というアルゴリズムを使い、
**Walk-Forward Validation**という方法で検証しています。これは「過去のデータだけで学習し、
まだAIが見ていない未来のデータでテストする」という、本番運用に近い厳密な検証方法です。

### グラフの読み方

| パネル | 見方 |
|---|---|
| ① フォールドごとの評価指標 | 5つの期間（フォールド）に分けて、それぞれでAIの的中率などを検証した結果。0.5（50%）のラインより上なら「コイン投げよりはマシ」 |
| ② 特徴量重要度 | AIが判断する際、どの指標を重視していたか |
| ③ 価格＋予測確率 | 紫の線が「AIが利確到達と予測した確率」。1に近いほど自信を持って「上がる」と予測 |
| ④ 混同行列 | AIの予測が当たったか外れたかの集計表 |
| ⑤ 予測確率の分布 | 自信を持って予測した時ほど当たっているか（緑が多いほど良い） |

### チェックポイント
- ①のグラフで、ROC-AUC（紫の棒）が複数のフォールドで0.5を下回っている場合、
  **そのモデルは「コイン投げ以下」**ということになります。精度がまだ実用レベルに
  達していないサインなので、特徴量の見直しやデータ量の追加を検討しましょう。
- ②の特徴量重要度がほぼ0の指標ばかりであれば、その特徴量はモデルにとってあまり
  役に立っていない可能性があります。逆に上位に来ている指標は、その銘柄・期間の
  値動きを説明する上で重要な手がかりだったと言えます。
- ⑤の分布で、予測確率が高い（1に近い）/低い（0に近い）ところに緑（正解）が
  偏っていれば「AIが自信を持っている時ほど当たっている」良いモデルと言えます。
  逆に赤（不正解）が均等にばらけている場合は、モデルの自信度があまり信頼できない
  状態です。

> ⚠️ **大切な心構え**：株価の値動きはそもそも予測が非常に難しく、プロのファンドでも
> 簡単には勝てない領域です。AIの的中率がコイン投げ程度、あるいはそれ以下になることは
> 珍しくありません。「AIに任せれば自動で儲かる」と考えるのではなく、**AIの限界を
> 正しく認識した上で、判断材料の一つとして使う**姿勢が重要です。

## ステップ5：人間にわかりやすい3段階の判定に変換（`step5_chart.png`）

### 何をしているか
ステップ4のAIはブラックボックスで「なぜそう判断したか」が説明しにくいため、
ステップ5では、あえて**人間が理解できるシンプルなルール**で「強気・警戒・中立」の
3段階に色分けし直しています。

| シグナル | 意味 | 判定条件（やさしく言うと） |
|---|---|---|
| 🟢 強気 | 買い場の可能性 | 売られすぎから値上がりに転じ始めていて、移動平均線からの乖離も縮まってきている時 |
| 🔴 警戒 | 様子見推奨 | 普段のボラティリティを大きく超えて急騰している（過熱）時 |
| ⚪ 中立 | どちらでもない | 上記どちらにも当てはまらない、平常運転の状態 |

### グラフの読み方

| パネル | 見方 |
|---|---|
| ① 価格＋シグナル | 緑▲＝強気判定が出た日、薄い赤の帯＝警戒判定が出ていた期間 |
| ② シグナル別件数 | 強気・警戒・中立がそれぞれ何日あったか |
| ③ シグナル別勝率 | 各シグナルが出た時、実際にステップ3の「利確到達」になった割合 |
| ④ シグナル別平均リターン | 各シグナルが出た時の平均損益 |

### チェックポイント
- ③のシグナル別勝率は、**「警戒」の勝率が「強気」より低くなっているか**を必ず確認しましょう。
  本来は警戒（見送り推奨）の勝率の方が低くなるのが設計の意図ですが、もし**警戒の勝率が
  強気を上回っている**場合、それは重要なシグナルです。
- そのようなズレが起きる典型的なケースは、相場が**強いモメンタム（勢いが続く）相場**の時です。
  「過熱しているから危ない」という逆張り的な考え方より、「勢いがある時はそのまま伸びやすい」
  という順張り的な値動きの方が実際の相場に合っていた、ということを示唆します。
- 逆に「中立」の勝率や平均リターンが極端に低い場合は、シグナルが出ていない（判断材料が
  少ない）局面ほど方向感が読みにくかった、と解釈できます。

> 💡 **初心者向けポイント**：株のルールは「いつでもどんな銘柄にも通用する万能なもの」ではありません。
> 同じ「過熱だから危ない」というルールでも、相場の性質（急騰相場か、レンジ相場か）によって
> 結果が逆になることがあります。シグナル別の勝率・リターンを確認する作業は、**そのルールが
> 今回のデータに対して本当に機能していたかを検証する**、とても重要なステップです。

## まとめ：各ステップの成果物をどう活用するか

| ステップ | 出力ファイル | 主な使い道 |
|---|---|---|
| 1 | `step1_dataset.csv` / `step1_chart.png` | 価格の大きな流れと、ボラティリティの推移を把握する |
| 2 | `step2_dataset.csv` / `step2_chart.png` | 「今が過熱しているかどうか」を統計的に確認する |
| 3 | `step3_dataset.csv` / `step3_chart.png` | 過去のシミュレーション結果から、戦略そのものが機能していたかを検証する |
| 4 | `step4_*.csv` / `step4_chart.png` / `step4_model.pkl` | AIの予測精度を客観的に評価し、過信していないか確認する |
| 5 | `step5_dataset.csv` / `step5_chart.png` | ルールベースの判定が実際の相場に合っていたかを検証する |

このプロジェクトは「AIで簡単に儲かる必勝法」を作るものではなく、**株価データから何が読み取れて、
何が読み取れないのかを、誠実に検証するためのもの**です。各ステップのグラフを見る際は、
「良い結果が出ているか」だけでなく「そのロジックが本当に機能しているか」という視点を
持つことが、何より大切なポイントです。
