# CSV出力ファイル 列定義ガイド

各ステップが出力するCSVファイルの列名と、その意味を一覧にまとめています。
どの列が何を表しているかわからなくなった時の参照用としてご利用ください。

---

## 共通事項

- インデックス列（一番左）は **日付**（`YYYY-MM-DD`形式）
- 数値は基本的に小数点以下も含む浮動小数点数
- フラグ系の列（`_flag`）は `0`（条件未成立）または `1`（条件成立）の2値

---

## step1_dataset.csv（ステップ1出力）

日足データに週足トレンド指標・テクニカル指標を加えたデータセット。
このCSVがその後の全ステップの土台になります。

### 価格・出来高（生データ）

| 列名 | 単位 | 内容 |
|---|---|---|
| `open` | 円 | 始値（その日の最初の取引価格） |
| `high` | 円 | 高値（その日の最も高かった価格） |
| `low` | 円 | 安値（その日の最も低かった価格） |
| `close` | 円 | 終値（その日の最後の取引価格） |
| `volume` | 株 | 出来高（その日に売買された株数） |

### ボラティリティ指標

| 列名 | 単位 | 内容 |
|---|---|---|
| `atr14` | 円 | ATR（14日間の平均的な値動き幅）。大きいほど値動きが荒い |
| `bb_pct_b` | — | ボリンジャーバンド %B。1を超えると上バンド越え（過熱気味）、0未満で下バンド割れ（売られすぎ） |
| `range_vs_atr` | 倍 | 当日の高安幅をATRで割った値。1を大きく超えると「普段より値動きが激しい日」 |

### モメンタム・トレンド指標

| 列名 | 単位 | 内容 |
|---|---|---|
| `rsi14` | 0〜100 | RSI（14日間）。70超で買われすぎ、30未満で売られすぎの目安 |
| `ma25` | 円 | 25日移動平均線 |
| `ma75` | 円 | 75日移動平均線 |
| `dev_ma25` | 比率 | MA25からの乖離率。正=MA上方、負=MA下方（例: 0.05 = MA比+5%） |
| `dev_ma75` | 比率 | MA75からの乖離率 |
| `mom5` | 比率 | 5日間リターン（5日前の終値比の変化率） |
| `mom20` | 比率 | 20日間リターン |
| `mom_accel` | 比率 | モメンタム加速度（`mom5 - mom20`）。正=短期の勢いが長期を上回っている |

### ローソク足（ヒゲ）指標

| 列名 | 単位 | 内容 |
|---|---|---|
| `upper_wick_ratio` | 0〜1 | 高安幅に対する上ヒゲの長さの比率。大きいほど上値を抑えられた |
| `lower_wick_ratio` | 0〜1 | 高安幅に対する下ヒゲの長さの比率。大きいほど下値を拾われた |
| `upper_wick_body_ratio` | 倍 | 実体に対する上ヒゲの長さの比率 |
| `lower_wick_body_ratio` | 倍 | 実体に対する下ヒゲの長さの比率 |
| `candle_dir` | 1 / -1 | 陽線（終値≥始値）=`1`、陰線=`-1` |

### 週足トレンド指標

| 列名 | 単位 | 内容 |
|---|---|---|
| `weekly_ma5` | 円 | 週足の5週移動平均線 |
| `weekly_ma13` | 円 | 週足の13週移動平均線 |
| `weekly_ma5_slope` | 比率 | 週足MA5の傾き（前週比変化率）。正=上昇トレンド、負=下降トレンド |
| `weekly_ma13_slope` | 比率 | 週足MA13の傾き |
| `weekly_trend` | 1 / -1 | 週足トレンドの方向。`weekly_ma5 > weekly_ma13` なら `1`（上昇）、そうでなければ `-1`（下降） |

## step2_dataset.csv（ステップ2出力）

ステップ1の全列に加え、ドメイン知識に基づく過熱感・ダイバージェンス指標を追加。

### 乖離率のZスコア（統計的な異常度）

| 列名 | 単位 | 内容 |
|---|---|---|
| `dev_ma25_zscore` | σ | MA25乖離率を過去60日間の分布で標準化した値。`±2`を超えると統計的な異常値 |
| `dev_ma75_zscore` | σ | MA75乖離率のZスコア |
| `oversold_flag` | 0 / 1 | `dev_ma25_zscore ≤ -2` のとき `1`（統計的に売られすぎ） |
| `overbought_flag` | 0 / 1 | `dev_ma25_zscore ≥ +2` のとき `1`（統計的に買われすぎ） |

### RSIダイバージェンス

| 列名 | 単位 | 内容 |
|---|---|---|
| `rsi_bullish_divergence` | 0 / 1 | 過去20日の安値圏で「価格は下値更新したがRSIは切り上がった」状態を `1` で示す（反転の兆し） |
| `rsi_divergence_strength` | ポイント | ダイバージェンス検出時のRSI切り上がり幅（強さの目安）。非検出時は `0` |

### ATR正規化による過熱感

| 列名 | 単位 | 内容 |
|---|---|---|
| `atr_dev_ma25` | 倍 | MA25からの乖離幅をATRで割った値。正方向に大きいほど「ボラティリティに対して急騰している」 |
| `is_overheated` | 0 / 1 | `\|atr_dev_ma25\| ≥ 2` のとき `1`（過熱状態） |
| `is_overbought_heat` | 0 / 1 | `atr_dev_ma25 ≥ +2` のとき `1`（急騰方向の過熱） |
| `is_oversold_heat` | 0 / 1 | `atr_dev_ma25 ≤ -2` のとき `1`（急落方向の過熱） |
| `pump_temporary_score` | — | 急騰の一時性スコア（`atr_dev_ma25`の正方向成分 × `mom_accel`の正方向成分）。値が大きいほど「過熱しながら加速中＝続きにくい可能性」 |
| `is_likely_temporary_pump` | 0 / 1 | `is_overbought_heat=1` かつ `mom_accel > 0` のとき `1`（一時的急騰の疑い） |

## step3_dataset.csv（ステップ3出力）

ステップ2の全列に加え、トリプルバリア法によるラベルとリスク評価列を追加。

### トリプルバリア法のラベル

| 列名 | 単位 | 内容 |
|---|---|---|
| `tb_label` | 0 / 1 | この日にエントリーした場合の結果ラベル。`1`=利確到達（成功）、`0`=それ以外 |
| `tb_barrier` | 文字列 | 到達したバリアの種類。`take_profit`（利確）/ `stop_loss`（損切り）/ `time_out`（期間切れ） |
| `tb_days_to_touch` | 日 | エントリーからバリア到達までの営業日数 |
| `tb_return` | 比率 | バリア到達時のリターン（例: `0.10`=+10%、`-0.05`=-5%） |
| `tb_barrier_mode` | 文字列 | バリアの設定方法。`atr`=ATR倍率モード、`fixed_pct`=固定%モード |

### 見送り判定（フォワードドローダウン）

| 列名 | 単位 | 内容 |
|---|---|---|
| `forward_max_drawdown` | 比率 | エントリー後の保有期間中における最大含み損（負の値。例: `-0.05`=-5%） |
| `avoid_entry_flag` | 0 / 1 | 保有期間中に一定割合（デフォルト3%）以上の含み損が発生した場合に `1`（見送り推奨） |

## step4_fold_metrics.csv（ステップ4: フォールド別評価指標）

Walk-Forward Validationの各フォールドにおける評価スコアを記録したファイル。

| 列名 | 内容 |
|---|---|
| `fold` | フォールド番号（1〜5） |
| `train_start` / `train_end` | 学習データの期間（開始日・終了日） |
| `test_start` / `test_end` | 検証データの期間（開始日・終了日） |
| `n_train` / `n_test` | 学習・検証それぞれのデータ件数 |
| `accuracy` | 正解率（全件中何件当たったか） |
| `precision` | 適合率（「1と予測した」中で実際に「1」だった割合） |
| `recall` | 再現率（実際の「1」をどれくらい拾えたか） |
| `f1` | F1スコア（precisionとrecallの調和平均。1に近いほど良い） |
| `roc_auc` | ROC-AUC（0.5=コイン投げ相当、1.0=完璧。0.5を大きく下回ると予測力なし） |

## step4_feature_importance.csv（ステップ4: 特徴量重要度）

AIが判断に使った各特徴量の貢献度を記録したファイル。
全フォールドの平均値で降順に並んでいます。

| 列名 | 内容 |
|---|---|
| （インデックス） | 特徴量名 |
| `importance` | 重要度スコア。値が大きいほどAIがその特徴量を重視していた |

---

## step4_oos_predictions.csv（ステップ4: Out-of-Sample予測結果）

検証期間（AIが学習に使っていないデータ）における予測結果を日付付きで記録したファイル。
各行が「この日に実際にモデルを動かした場合の予測」に相当します。

| 列名 | 内容 |
|---|---|
| `fold` | この予測がどのフォールドの検証データに属するか |
| `y_true` | 正解ラベル（ステップ3の`tb_label`と同じ。`0` or `1`） |
| `y_pred` | モデルの予測ラベル（`0` or `1`） |
| `y_proba` | モデルが「`1`（利確到達）」と予測した確率（0〜1）。0.5以上で`y_pred=1`になる |

## step5_dataset.csv（ステップ5出力）

ステップ3の全列に加え、3段階アシストシグナルの判定結果と中間フラグを追加。

### シグナル判定の中間フラグ

| 列名 | 内容 |
|---|---|
| `rsi_reversal_flag` | RSI反転条件が成立している場合に `1`（売られすぎ圏からの切り返し、またはダイバージェンス成立） |
| `deviation_shrink_flag` | MA乖離が縮小中（下方乖離からゼロに向かっている）の場合に `1` |
| `overheat_warning_flag` | ATR正規化乖離が過熱水準（`is_overbought_heat=1`）の場合に `1` |

### 最終シグナル

| 列名 | 内容 |
|---|---|
| `assist_signal` | 3段階シグナルの最終判定。`強気`（買い場の可能性）/ `警戒`（様子見推奨）/ `中立`（どちらでもない）の3値 |

> **優先順位**：`overheat_warning_flag=1` の日は、`rsi_reversal_flag` と `deviation_shrink_flag` が
> 両方成立していても `警戒` が優先されます（過熱中の買いエントリーを安全側に倒す設計）。
